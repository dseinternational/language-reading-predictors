## Step 1 — reproducible interaction screen (`scripts/screen_interactions.py`)

LightGBM (`objective="mae"`, 300 trees, lr 0.03) + `shap.TreeExplainer`
`shap_interaction_values`, ranked by mean |off-diagonal|. SHAP on the full fit;
GroupKFold (by `subject_id`) only for a reported CV-MAE. Run for `ewrswr`
(level) and `ewrswr_gain` (gain) over the canonical registry predictor sets
`Predictors.DEFAULT_LEVEL` / `DEFAULT_GAIN` (broader than the earlier note's
focused lists, so rankings re-rank — by design, to avoid hand-picking). Tables
written to `output/interaction_screen/{level,gain}/`.

Two points worth recording:

- **Target-leakage guard.** `DEFAULT_LEVEL` contains `ewrswr` itself; the screen
  drops the target column from the predictors (for gain the `ewrswr` *baseline*
  legitimately stays, since the target is `ewrswr_gain`).
- **`spphon` dominates the level screen** (mean |main| ≈ 4.5, then `yarclet`
  ≈ 3.7). Phonetic spelling is itself a decoding/reading outcome, highly
  collinear with reading level; the earlier focused screen excluded it. With it
  in, the top interactions are `erbword × spphon`, `aptinfo × spphon`,
  `nonword × yarclet`, then `celf × yarclet`, `b1exto × yarclet`,
  `spphon × yarclet` — letter-sound remains a hub with concepts/decoding/vocab.
- **Gain** reproduces the earlier signal: `age` dominant main effect; top
  interactions `age × yarclet`, `b1exto × celf`, `age × blending`,
  `celf × attend`, `yarclet × attend` present (dose × readiness).

## Step 2 — LRP71: L → W moderated by expressive vocabulary (E = eowpvt)

Extends LRP58 (mechanism L → W, adjustment {G, A, E, R, W_pre}) with a **linear**
moderation, leaving the nonparametric `f_mech(L)` HSGP intact:

    η = … + f_mech(logit L_post) + gamma_mod·z(E) + gamma_int·z(logit L_post)·z(E)

Both coefficients use the regularising `gamma_cross` prior, Normal(0, 0.3); L and
E are standardised. `gamma_int > 0` would mean phonics converts to reading more
strongly for higher-vocabulary children (lexical-quality / dual-route).

**Machinery** (generic, so LRP70 drops in later): `build_mechanism_model` gains a
`moderator_symbol` arg (keep-mask for M, standardised `z_L`/`z_M` computed on the
kept rows, `gamma_mod`/`gamma_int` added after the confounder loop). The pipeline
removes the moderator from the plain confounder loop so E is represented **only**
by its standardised main effect + interaction (no collinear duplicate `gamma_E`);
the standardised main effect still satisfies the DAG E-adjustment. With the
interaction present, the `f_mech` curve is the L → W slope **at mean E**.

### Result (reporting config; n = 157 transition-rows, 53 children)

| coefficient | mean | 95% CI (eq-tailed) | P(>0) |
| --- | ---: | --- | ---: |
| `gamma_int` (moderation) | −0.091 | [−0.193, 0.011] | 0.04 |
| `gamma_mod` (vocab main effect) | +0.168 | [0.029, 0.311] | 0.99 |

Sampling pristine: all R-hat = 1.00, ESS in the thousands–tens-of-thousands, no
divergences.

**Read honestly:** the moderation is **suggestive, not credible** — the 95% CI
spans zero, leaning slightly *negative* (if anything, higher-vocabulary children
convert letter-sound to reading marginally *less* strongly, not more). The vocab
*main* effect is credibly positive, as expected. This is a legitimate,
publishable result: within this sample, phonics converts to reading largely
**independently of vocabulary level**. The value of LRP71 is establishing the
interaction machinery and testing the dual-route hypothesis honestly.

**PSIS-LOO vs LRP58** (`output/statistical_models/comparison/mechanism_loo_compare.csv`,
reporting; both models share the same 157 rows so `az.compare` is like-for-like):
LRP71 ranks first but elpd_diff ≈ 1.0 with dse ≈ 1.4 — i.e. within noise. Adding
the moderation does **not** meaningfully improve predictive fit, consistent with
the non-credible `gamma_int`.

### LRP70 (celf moderator) — reserved, deferred

The screen ranks `celf` (concepts, symbol F) highest, but LRP58's DAG treats F as
a **descendant of L**, so conditioning on it (as a main effect + interaction)
risks collider bias and attenuation of `f_mech` itself. We therefore built the
DAG-clean `eowpvt` model (LRP71) first and **reserved LRP70** pending a DAG review
of whether/how to admit celf (e.g. a moderator-only formulation, or revisiting
F's position in the DAG). The factory already supports it — adding LRP70 is one
module once the DAG question is settled.

## Incidental: arviz 0.x → 1.x migration (pre-existing breakage)

The Bayesian pipeline (all of LRP52–60, not just LRP71) did **not** run on the
arviz version the repo pins (`arviz>=1.0` → 1.1.0; likely since the dependency
bump in `e50d201`). The diagnostics/reporting code used the removed arviz 0.x
API. Migrated, minimally and behaviour-preserving:

- `ELPDData.elpd_loo`/`.p_loo`/`.looic` → `.elpd`/`.p` (+ `se`); `az.compare`
  `elpd_loo` column → `elpd` (`pipeline.py`, `reporting.py`,
  `fit_statistical_model.py`, `compare_statistical_models.py`).
- Plotting moved to `arviz_plots` (PlotCollection `.savefig`); dropped removed
  kwargs (`combined`, `kind`, `divergences`); `plot_posterior`→`plot_dist`,
  `plot_ppc`→`plot_ppc_dist`. Each plot is now guarded so an API/backend hiccup
  warns rather than aborting the fit (`diagnostics.py`, `pipeline.py`).
- `DataTree.groups` is a property of `/`-prefixed paths in 1.x (was a method of
  bare names) — fixed the group check in `mechanism_loo_compare`.
- Minor: guarded `tau_forest` against a missing joint run so the comparison
  script completes when only a subset of models is fitted.

LRP58 (regression guard) and LRP71 both now fit end-to-end and render.
