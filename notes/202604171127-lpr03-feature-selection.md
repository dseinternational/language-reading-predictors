## Motivation

LRP03 predicts expressive-vocabulary **gains** (`eowpvt_gain`) between
consecutive longitudinal timepoints, paralleling LRP01's task on
word-reading gains. Baseline started from the full
`Predictors.DEFAULT_GAIN` set plus the auto-included `eowpvt` base
variable (34 predictors total), n=161, with no outlier exclusion.

Following the LRP02 playbook, we first tune MAE hyperparameters on
the full predictor set before starting any feature-selection work.
The MAE objective keeps importance rankings from being dominated by
a few high-absolute-gain observations and pairs naturally with
median-absolute-error (MedAE) evaluation.

The tuner settled on a **very conservative ensemble**: only 16 trees,
aggressive learning rate, narrow (7-leaf) trees, and strong L1
regularisation. This is consistent with n=161 and a high-variance
signed target — the model can't afford many trees before
over-fitting.

### Baseline-under-defaults vs MAE-tuned (test config, 10-fold GroupKFold)

|---|---|---|

| In-sample MAE | — | — |
| **In-sample R²** | **0.983** | **0.365** |

Every CV metric improves substantially. **CV R² flips from negative
to positive** — the untuned model was generalising worse than the
mean-baseline predictor, which is now fixed. In-sample R² drops
from 0.983 to 0.365; the tuned model no longer memorises its
training data.

The gap between in-sample R² (0.365) and CV R² (0.045) remains
large — indicating genuine predictive difficulty at n=161 — but
it's a "modest signal" gap rather than an "over-fit" gap.

- **A further ~7 features below 0.005** are at or near the

## Candidates for Select01 (next step)

At this 16-tree MAE-tuned state, about **half the predictor set is
at or near zero importance**. Two loose tiers for the first
feature-selection cut:

### Tier A — strictly zero importance (safe drops)

`yarcsi`, `spphon`, `nonword`, `hearing`, `earinf`, `numchil`,
`gender`. Seven features.

### Tier B — very low importance (0.001 – 0.005)

`erbnw`, `time`, `group`, `blending`, `vision`, `area`, `dadedupost16`,
`mumedupost16`, `erbword`, `attend`, `behav`. Eleven features on the
edge of the noise floor.

A conservative Select01 would drop the 7 Tier-A features, retune,
and re-inspect. A more aggressive cut would drop all 18 Tier-A + B
features (34 → 16 predictors in one step).
