# Uniform gradient-boosting feature selection across all GB models

**Date:** 2026-06-21
**Scope:** all 22 GB primaries (LRP01‚ÄìLRP22) re-derived from the full default
predictor set by one consistent method, replacing the previous patchwork
(LRP01‚Äì10 hand-curated; LRP11‚Äì22 replication-method; LRP08/09 under-pruned). The
exploratory transform/experiment variants on LRP01/02/04 are retired; same-skill
`_noconstruct` variants are regenerated. Supersedes the per-model selections in
`notes/202604*` and `notes/202606201500-gb-replication-findings.md` (kept as the
historical record) and the LRP11‚Äì22 / same-skill notes
(`202606210900`, `202606210930`).

## Why

The replication (`202606201500`) showed the suite's selections were a mix of
methods and that several models still carried correlated-predictor redundancy
(LRP02/03/05/07/08/09: 1‚Äì8 distance-correlation ‚â• 0.70 pairs). This pass applies
**one reproducible algorithm** to every model so the whole suite is consistent
and redundancy-free, and so the selection provenance is a single dated step
rather than an accreted history.

## Method (applied to every primary, from the full set)

For each model, starting from the full default predictor set (`DEFAULT_GAIN`,
34, for gain models; `DEFAULT_LEVEL` minus target, 32, for level models), using
the deterministic full-set out-of-fold permutation importance and
distance-correlation matrix (seed 47):

1. **Redundancy filter** ‚Äî rank predictors by full-set OOF permutation
   importance; drop a predictor if its distance-correlation with an
   already-kept, higher-importance predictor is ‚â• 0.70. Gain models **force-keep
   the baseline** measure (the regression-to-the-mean anchor).
2. **Noise-floor cut** ‚Äî drop remaining predictors with OOF importance ‚â§ 0.005.
3. **Standardised-instrument swap** ‚Äî prefer the standardised test over its
   bespoke taught sibling (`eowpvt`‚Üê`b1exto`, `rowpvt`‚Üê`b1reto`) when it does not
   reintroduce a ‚â• 0.70 pair.
4. **Re-tune** the reduced set (Optuna 150-trial MAE, 10-fold GroupKFold,
   seed 47); pin the params.

Encoded as a single `SelectionStep` per module (`date="2026-06-21"`); every
model now has **0 distance-correlation ‚â• 0.70 pairs** among its selected
predictors.

## What changed

- **LRP01‚Äì10 re-derived.** The hand-curated, construct-driven drops are replaced
  by the algorithm. Because the filter keeps the highest-importance member of a
  correlated cluster, predictive same-skill measures previously dropped by hand
  (e.g. `lrp04` `b1exto`, `lrp10` `eowpvt`) return to the primaries and instead
  drive the same-skill variants. Gain models now force-keep the baseline
  (e.g. `lrp01` retains `ewrswr`; it had been dropped before).
- **Residual redundancy cleared** in LRP02/03/05/07/08/09 (the open "re-prune
  LRP08/LRP09" item from the replication note is now closed).
- **LRP12 26‚Üí3** ‚Äî the uniform noise-floor now applies where it had been held
  back for this flat-importance target; CV cost is real (see table).
- **Transform/experiment variants retired** (LRP01: `_prediction`/`_log`/
  `_exp_*`; LRP02: 8 `_log`/`_quantile`/`_select`/`_prediction` variants; LRP04:
  `_quantile`). Preserved in git history; can be revisited.
- **Same-skill `_noconstruct` variants** regenerated for level models whose
  pruned primary keeps a same-skill sibling: `lrp04` (drop `b1exto`), `lrp18`
  (`aptinfo`), `lrp20` (`aptgram`), `lrp22` (`deappin` ‚Üí null). `lrp08` no longer
  needs one (uniform pruning already drops `b1reto`).

## Caveats

- Same exploratory standing as before: gain models are near-noise
  (baseline-driven regression to the mean); level-model R¬≤ is largely concurrent
  same-construct correlation (the `_noconstruct` variants expose this); reduced
  *rankings* are reliable only for the top one or two predictors; pinned
  hyperparameters are CV-equivalent optima, not unique.
- **LRP12** trades CV MAE (2.96‚Üí3.24) for the uniform rule on a flat-importance
  target ‚Äî a deliberate consequence of one method everywhere, not a tuning bug.
