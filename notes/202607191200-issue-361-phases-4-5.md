# Issue 361 Phases 4 and 5: teaching route and replication gates

## Decision

Use `lrp-rli-itt-001` as the canonical undergraduate route. It is the shortest complete path through a randomised bounded-count model: one outcome, one t1→t2 row per child, an immutable typed plan, no cross-baselines, no child random intercept and no floor-rule branch. The teaching chapter must say explicitly that its causal licence belongs only to the randomised assignment contrast in the available-case analysis population. Baseline and age coefficients are precision associations. Partial pooling is taught as a separate repeated-measures extension because adding a child random intercept to a one-row-per-child ITT model would be unidentifiable and would misdescribe the actual fit.

## Output publication

Replace the destructive start-of-fit reset with a transaction. Each model×configuration run writes every artefact to a hidden sibling directory; the shared final report stage promotes it only after key findings and report files have succeeded. If an earlier successful directory exists it is moved to a private backup immediately before promotion and restored if promotion fails. A failed fit therefore preserves the last complete publication and cannot leave a report that mixes stale and partial files.

Transaction creation best-effort removes matching staging directories older than seven days, while leaving recent siblings alone because they may belong to concurrent fits. A stale private backup is removed only when the final publication exists; without that final directory the backup may be the only recoverable successful fit. The staging directory's access mode is derived through a temporary directory created under the process umask, avoiding both `mkdtemp`'s fixed `0700` mode and the process-wide race that would result from reading the umask by changing it.

## Exact environment record

Write `environment-lock.json` beside every successful fit. The lock records exact conda package names, versions, builds, channels and available hashes plus every installed Python distribution and sanitised direct-install metadata. `config.json` records the lock filename and SHA-256 digest. The lock complements rather than replaces `environment.yml`: the YAML declares the supported environment, while the lock identifies the concrete environment that produced one publication.

## Incremental gates

Add a fixed-seed synthetic Beta-Binomial smoke test that launches an isolated process with writeable compiler caches, compiles the same canonical ITT equation through PyMC and nutpie, draws a tiny posterior and posterior predictive, and checks dimensions, finite draws and score bounds. This is deliberately a smoke gate, not an inferential result and not a substitute for reporting-size convergence checks.

Begin static type checking at the boundaries made explicit by Phases 2–5: the typed ITT declaration and resolved plan, shared sampling/report stages, output transaction, provenance/environment lock and registry-documentation contract. Keep `follow_imports = "skip"` and a named file list so the first gate is strict without pretending the remaining dynamic family implementations are already typed. Extend the list as each family leaves the `ModelSpec.extra` compatibility boundary.

Store a generated `docs/models/registry-counts.json` snapshot sourced from `definitions.KINDS`, `definitions.MODEL_REGISTRY` and `registry.discover_models()`. CI fails with a regeneration command when any total changes. Prose should still avoid hand-maintained totals; the snapshot exists to make intentional registry changes reviewable and silent drift impossible.

## Non-goals

These phases do not change an estimand, prior, likelihood, sampling preset or fitted equation. They do not complete the remaining Phase 2 typed-family migration or Phase 3 family-module split. Those are separate structural tranches and should not be hidden inside the teaching and replication change.
