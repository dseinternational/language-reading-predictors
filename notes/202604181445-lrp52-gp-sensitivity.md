# LRP52 HSGP sensitivity — HSGPs off becomes the ITT default

> [!WARNING]

Date: 2026-04-18

## Decision

HSGPs on age and own baseline are **disabled by default** in the ITT models
(LRP52, LRP53, LRP54). The `use_age_gp` and `use_own_baseline_gp` flags remain
wired through the factory so the GPs can be re-enabled per outcome if the
posterior predictive check shows unmodelled nonlinearity.

## Evidence

Three sequential LRP52 reporting-config fits were run (6 chains × 6000 tune ×
6000 draws each) on the full ITT sample (53 children after dropping 1 missing
baseline).

## Diagnosis

Across both GP-on priors, the HSGP amplitude posteriors had 95 % CIs that
touched zero:

| Amplitude | HN(1.0) 95% CI | HN(0.3) 95% CI |
|---|---|---|
| `f_A__eta` | (0.000, 0.577) | (0.000, 0.346) |
| `f_ypre__eta` | (0.000, 0.687) | (0.000, 0.389) |

When η can be near zero, the product η × `g_unit` (with `g_unit` the 20-basis
coefficient vector) is degenerate: any `g_unit` values produce the same
`f_A`, so the posterior over `g_unit` is only weakly identified. This is the
Neal's-funnel geometry between η and the basis weights. NUTS cannot step out
of a funnel by reducing step size — hence the invariance between
`target_accept = 0.95` and `target_accept = 0.98`.

The tighter HN(0.3) prior halved the divergences but did not eliminate them
because η = 0 remains admissible. Turning the GP off removes the funnel by
construction.

## Why this does not weaken the model

- **Sample size**: 53 children with one post-score each cannot identify a
  20-basis HSGP. p_loo says the GP was contributing ~1.7 effective parameters
  over and above the linear model — barely more than basis noise.
- **Flexibility is already present**: the linear predictor includes own
  baseline (γ_own) and seven cross-baseline couplings. These absorb almost
  all explainable variance (γ_own CI (0.45, 0.88), dominating all cross
  couplings).
- **τ is unchanged** between every variant. The science does not depend on
  the GP terms.

## Policy

- **ITT models (LRP52/53/54)**: `use_age_gp=False`, `use_own_baseline_gp=False`
  by default. Per-model override available via the `extra` dict on the
  `ModelSpec` if PPC reveals nonlinear tension.
- **Joint model (LRP55)**: age GP partial pooling (`mu_A` + tight
  `delta_A_k`) kept. LRP55 has 8 × 53 = 424 observations, so the shared
  `mu_A` is identifiable even though each outcome-specific `delta_A_k` has
  only 53 observations worth of signal.
- **Mechanism models (LRP56/57/58)**: mechanism GP `f_mech` kept — it is the
  estimand, and all three phases are stacked (~159 obs) so the GP is
  identifiable.

Re-run any ITT model with GPs on by passing a modified `ModelSpec`:

```python
from language_reading_predictors.statistical_models.context import ModelSpec
from language_reading_predictors.statistical_models.pipeline import fit_itt

spec = ModelSpec(
    model_id="lrp53_gp",
    kind="itt",
    title="LRP53 sensitivity: receptive vocab with HSGPs on",
    outcome_symbol="R",
    extra={"use_age_gp": True, "use_own_baseline_gp": True},
)
ctx = fit_itt(spec, config="reporting")
```

## Tightened prior retained for the joint and mechanism models

Although the GPs are off for the ITT models, `priors.eta_main_prior()` was
also tightened from `HalfNormal(1.0)` to `HalfNormal(0.3)` during this
investigation. The tighter prior is kept — LRP55 and LRP56/57/58 inherit it.
The rationale is documented in the docstring of `eta_main_prior()`.
