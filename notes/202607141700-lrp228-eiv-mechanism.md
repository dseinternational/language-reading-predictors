# Errors-in-variables code→word-reading mechanism (#228 item 14)

Date: 2026-07-14

Related: #228 (suite-gap analysis, item 14), #13 (mm-001 reparameterisation — done), mech-058 (the L→W mechanism), mm-001 (correlated-domain-factor measurement model), METHODS.md (the "we do not separately model measurement error" leak).

## What #14 asked, and what was already there

`METHODS.md` names measurement error as an unmodelled leak in the L→W mechanism: mech-058 puts the **observed** letter-sound logit into an HSGP, so the association is attenuated by the noise in the letter-sound test. #14 asks for the correction, with "the repaired mm-001 machinery" as the vehicle.

The exploration found that **mm-001 already _is_ an errors-in-variables regression**: its structural leg regresses word reading on a latent **code factor built from L+B** (2 indicators → measurement error identified), so mm-001's `β_code` is already a measurement-error-corrected code→reading slope. #13 (mm-001's reparameterisation) is done and mm-001 converges. A pure single-indicator L→W EiV is **not** cleanly identifiable at n≈54 (YARC-LSK is one measure; it would need an external reliability, not in the repo).

## What LRPMM02 adds

LRPMM02 gives the piece mm-001 does not: the EiV counterpart of the **mech-058 estimand**. mm-001 adjusts the code slope for the co-varying latent vocabulary and grammar factors; LRPMM02 instead adjusts for the **mech-058 confounder set** (G, age, W baseline, hearing, speech), so `β_code` is directly comparable, head-to-head, with mech-058's attenuated observed-L association — the explicit #14 "how much does correcting for measurement error move the slope?" read.

**Identification (drove the design).** A standalone 1-factor/2-indicator code model (L+B alone) is under-identified. So LRPMM02 keeps mm-001's full, identified 3-factor measurement model (code {L,B} / vocab {R,E} / grammar {F,T}) but **isolates the structural leg to the code factor only**. The vocab/grammar factors stay in the measurement model purely to identify the code factor.

**Frame caveat.** The corr_factor measurement model is between-child (`phase_mode="span"`, t1 → t4), not mech-058's phase-stacked within-transition frame — a close companion, not an identical re-fit. Intervention dose (`attend`) is omitted (per-period collider, no clean span analogue; t1 attendance is 0).

## The opt-in factory extension

`build_correlated_factor_model` gains two opt-in flags (defaults reproduce mm-001/101 **byte-identical** — the corr_factor prior-defaults regression test passes, and the default structural-leg branch is unchanged):

- `structural_factors: tuple | None` — when set (e.g. `("code",)`), the structural leg regresses on that factor subset (`beta_factor` on a `struct_domain` coord) while the full measurement model is unchanged.
- `use_group: bool` — adds the randomised arm `G` as an adjusted-association covariate (association-scale prior, not tau).

`fit_correlated_factor` reads both from `spec.extra`, loads the mech-058 raw covariates (hearing/speech via the missing-indicator path), and the structural-slope summary handles the `struct_domain` coord + `beta_G`.

## Reporting-tier read

`β_code = +0.331` (logit; 95% CrI [+0.027, +0.640]; P(>0) = 0.984) — **larger** than mm-001's `+0.27`, the direction attenuation correction predicts (removing measurement noise inflates the slope toward its true value), and a positive, credible measurement-error-corrected code→reading association. Side-result: `β_age = −0.342` (P(>0) = 0.001) — younger children show larger reading gain, consistent with adj-065 / gc-085.

**Convergence (honest flag).** 0 divergences; **the headline `β_code` (R̂ = 1.0004, ESS = 9,026) and every reported deliverable — the structural covariates (ESS 33k–37k) and all six loadings/communalities (ESS 14k–22k) — converge cleanly.** The gate is missed only on **3 factor-correlation Cholesky elements** (`factor_cov[3,4,5]`, R̂ ≤ 1.027, ESS ≥ 160): isolating the structural leg to the code factor removes the outcome's information from the vocab/grammar factors, so their correlations mix a little worse. Those correlations are **identification nuisance here, not this model's deliverable** — mm-001 is the model of record for the factor-correlation matrix. So mm-002 is usable and gate-flagged, in the same spirit as the divergence-only / nuisance-miss fits the refit reports as "usable". The corrected-vs-attenuated comparison against mech-058's observed-L association folds into the model chapter + the next full-suite refit.
