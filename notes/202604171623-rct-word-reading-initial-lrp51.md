# LRP51 — Bayesian beta-binomial DAG for word-reading gain (initial design)

## Purpose

LRP51 introduces the first **Bayesian** model in the project, alongside the
existing tree-based family (LRP01–LRP06). It is the start of a parallel
modelling track that uses generative, causally-structured models to
complement the predictive-focus tree models.

Specifically, LRP51 estimates word-reading progress from timepoint t1 to
timepoint t2 (EWRSWR score) under an explicit directed acyclic graph (DAG)
of influences: age, receptive vocabulary, expressive vocabulary, letter
sound knowledge, intervention group, and intervention attendance.

Like all work in this project, LRP51 is exploratory and preliminary.

## DAG

```
A (age)
├── → R (receptive vocabulary, ROWPVT)
├── → E (expressive vocabulary, EOWPVT)
├── → L (letter sound knowledge, YARCLET)
└── → W (word reading, EWRSWR)
R → E
R → W
E → W
L → W
G (group, 2-level)  → W
I (attend sessions) → W
```

- A → R, A → E, A → L: age influences each language/pre-reading skill.
- A → W, R → W, E → W, L → W: age and each prerequisite skill influence
  word-reading gain directly.
- R → E: receptive vocabulary supports expressive vocabulary.
- G → W: intervention group (binary contrast).
- I → W: dose of intervention delivered across the t1–t2 interval.

All edges are measured at t1 (predictors) except the outcomes at each
node, which are measured at t2.

## Likelihoods

All four outcome nodes use **beta-binomial** with a node-specific item
total N. For the three intermediate measures (R, E, L), the current-timepoint
value at t2 is the outcome. For the primary outcome (W), we model
EWRSWR at t2 with EWRSWR at t1 carried as an offset on the linear
predictor (see "Word-reading parameterisation" below).

| Node | Outcome | N (trials) | Notes |
|---|---|---:|---|
| R (ROWPVT) | t2 raw score | 190 | ROWPVT-4 item bank |
| E (EOWPVT) | t2 raw score | 190 | EOWPVT-4 item bank |
| L (YARCLET) | t2 raw score | 32 | YARC letter-sound subtest (confirmed by observed max) |
| W (EWRSWR) | t2 raw score | 64 | EWRSWR item bank; confirm from test manual before reporting |

The N values for ROWPVT and EOWPVT assume the fourth editions; confirm
against the edition actually administered in this cohort before the
reporting run. The YARCLET N matches the observed maximum and is safe.
The EWRSWR N is provisional — observed max is 64, but the test may
admit higher scores; confirm before reporting.

**Why beta-binomial rather than Gaussian or negative binomial.**
Raw scores are bounded counts of items correct. Beta-binomial respects
that boundedness, naturally handles overdispersion (children are not
items of identical difficulty answered independently), and gives each
node the same functional form so the DAG composes cleanly. Gaussian
ignores the bounds and is heteroscedastic at floor/ceiling; negative
binomial is unbounded above and loses the "proportion known"
interpretation.

### Word-reading parameterisation

Gain (`ewrswr_gain = ewrswr_t2 − ewrswr_t1`) can be negative in this
cohort, which is incompatible with beta-binomial support. We therefore
model `k = ewrswr_t2` out of `N = 64` with `ewrswr_t1` entering the
linear predictor as a covariate (an offset in effect). This preserves
the direction of "gain given baseline" without censoring the rare
downward-moving observations.

An alternative formulation — model words-newly-learned out of
words-not-yet-known — would give a direct learning-rate interpretation
but silently drops negative-gain observations. We prefer faithfulness
to the data; the alternative may become `lrp51_learning_rate` variant.

## Linear predictor and HSGP structure

For each node Y with parents `pa(Y)`, the logit-scale linear predictor is:

```
logit(p_Y) = β0_Y + Σ_{X ∈ pa(Y)} [ βX·x_std + f_X(x_std) ]
```

where `f_X ~ HSGP(ExpQuad(ls=ell_X), m=M, L=L)` captures variation from
the central linear trend on each parent. One GP per edge; edges combine
additively. This is identifiable (each GP is centred, zero-mean prior),
cheap (1-D inputs), and composes naturally with the DAG.

**Edges with GPs** (9 total):

- A → R, A → E, A → L, A → W
- R → E, R → W
- E → W
- L → W
- I → W

**Edges without GPs:**

- G → W: 2-level binary contrast enters as a single linear coefficient.
  A hierarchical formulation is not justified at two levels (between-
  group variance is essentially unidentified); partial pooling is
  deferred to future models where a level with many units exists
  (e.g. sites, cohorts).

**HSGP hyperparameters** are set via
`pm.gp.hsgp_approx.approx_hsgp_hyperparams()` using an input-range-
and length-scale-range argument per edge. Starting defaults mirror
vocabulary-growth conventions:

- `ell_unit ~ InverseGamma(3, 3)` (shape-scale on the standardised
  input range)
- `eta ~ HalfNormal(0.4)` (GP amplitude)
- `m ≈ 20–40` basis functions; `L` set from input range with margin.

## Data scope for LRP51

- One row per subject, using the t1 → t2 transition only.
- Baseline (t1) predictors: age, rowpvt, eowpvt, yarclet, ewrswr, group,
  attend.
- Outcomes at t2: rowpvt, eowpvt, yarclet, ewrswr.
- No subject random effect in LRP51; subject-level hierarchy is deferred
  to subsequent models (LRP52+) that use repeat observations.

## Sampling and diagnostics

Three presets mirror the existing `RunConfig`:

| Preset | draws | tune | chains | target_accept | LOO |
|---|---:|---:|---:|---:|---|
| dev | 500 | 500 | 2 | 0.90 | no |
| test | 1000 | 1000 | 4 | 0.95 | no |
| reporting | 2000 | 2000 | 4 | 0.99 | yes |

Reporting-run acceptance criteria:

- Divergences < 1% of post-tuning draws.
- All `r_hat` ≤ 1.01.
- Bulk ESS ≥ 400 per parameter of interest.
- No LOO Pareto `k > 0.7` for more than a handful of observations.

Posterior-predictive checks compare simulated outcome distributions to
observed at each node. Mediation summaries (total / direct / indirect
effects of A on W via R, E, L) are computed from posterior draws.

## Code organisation

LRP51 introduces a Bayesian pipeline alongside the existing
`EstimatorPipeline` tree-model pipeline, reusing the `ModelConfig` /
`pipeline_cls` / `fit_model.py` dispatch pattern so the CLI does not
change. Proposed layout:

```
src/language_reading_predictors/models/
  bayesian/
    __init__.py
    definition.py           # BayesianModelDefinition, NodeSpec, EdgeSpec
    base_pipeline.py        # BayesianPipeline: prepare_data, build_model,
                            # sample, diagnostics, posterior_predictive,
                            # posterior_summary, plots, save_config, report
    hsgp.py                 # hsgp_edge() helper
    priors.py               # shared prior presets
    betabinom_pipeline.py   # BetaBinomialHSGPPipeline subclass
  lrp51.py                  # declares MODEL_LRP51 + registers
```

`BetaBinomialHSGPPipeline.build_model` walks nodes in topological
order, materialises HSGP terms for GP edges and linear betas otherwise,
and sums contributions into each child's linear predictor. Adding
a new Bayesian model is a matter of declaring its DAG; no new plumbing.

`RunConfig` gains `draws`, `tune`, `chains`, `target_accept` with the
presets above. Existing tree models ignore these fields.
